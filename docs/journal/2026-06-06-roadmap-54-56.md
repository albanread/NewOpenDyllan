# 2026-06-06 — Roadmap: Sprints 54–56, the last three rungs of front-end self-hosting

*A forward plan, not a retrospective. The ratified architecture
(`ARCHITECTURE.md` §Roadmap) already pins the direction; `DYLAN_SEMA_WIRE.md`
already names Sprint 54 by its hook (`lower_with_model`). This entry makes
54–56 concrete and records the one prerequisite that could quietly
undermine all three.*

## Where we are on the ladder

```
lexer   ✅ default
parser  ✅ default (Rust = fall-back + verify oracle)
macros  ✅ opt-in   (NOD_EXPAND_WITH_DYLAN)
sema    ◐ oracle done (dump-sema + DYLAN_SEMA_WIRE); Dylan recording walk WIP
lower   ⬜ still Rust  (AST → DFM)
        ── DFM IR ──  (the permanent front-end/back-end cut)
codegen/GC/JIT/linker  — permanent Rust, never migrates
```

Every rung uses the same move: **write the stage in Dylan → gate it
byte-identical against the Rust original via a wire format → flip it
load-bearing.** 54–56 are the last three rungs.

## The on-ramp (finish before 54 — small but load-bearing)

Two things must be solid first, because everything from 54 on routes
through the static-linked Dylan shim:

1. **Finish the 53.x Dylan sema walk.** The checkpointed
   `tests/nod-tests/fixtures/dylan-sema.dylan` (top-names mostly done;
   classes / slot accessors / generics remain), oracle-gated by
   `dump-sema`. Strip its DBG diagnostics and write the
   `sema_topnames.rs` gate it references.

2. **Fix the shim class-id drift (named pre-54 task, was P1-4).** The
   `aot.rs:1037` `ClassRedefinitionNotSupported` / class-id-drift assertion
   that also crashes `nod-driver eval` on the default shim path. **This is
   the keystone risk.** The migration's whole premise is that a Dylan
   stage runs via the shim and exchanges wire formats; if the shim's class
   registration isn't deterministic across the wire crossing, *no*
   shim-hosted stage (sema in 54, lowering in 55) can be trusted
   byte-identical. Fix it once, with a determinism test, before building
   on it.

(Also rides along cheaply somewhere in here: the `lexer_oracle`
parallel-build flake — serialise the build-shelling tests.)

## Sprint 54 — `lower_with_model`: make the Dylan sema authoritative

Flip sema from "verify-only oracle" to **load-bearing**: the host's
lowering consumes the Dylan-produced `SemaModel` (off the wire) instead of
recomputing sema in Rust. This is precisely what the sema wire contract
was built for (`DYLAN_SEMA_WIRE.md` names it).

- **Gate:** `dump-dfm` byte-identical whether the model came from the Rust
  sema or the Dylan walk.
- **Why it earns its own sprint:** it's the first time a Dylan stage
  *feeds* the back-end rather than just being compared to it. Class and
  symbol identity must survive the wire crossing exactly — which is why
  the class-id-drift fix is a hard prerequisite, not a nice-to-have.
- **Flag:** `--sema-with-dylan` / `NOD_SEMA_WITH_DYLAN`, fall-back to Rust
  sema, same discipline as the parser/expander flags.

## Sprint 55 — AST → DFM lowering in Dylan

Port the **last front-end stage** — lowering itself — to Dylan, emitting
the DFM wire. The DFM IR boundary is the permanent architectural cut, so
the output contract is already specified (`DFM.md`). When this lands,
**every front-end stage is Dylan-written.**

- **Gate:** `dump-dfm` byte-identical, Dylan-vs-Rust, across the corpus.
- **Expect sub-phasing** — lowering is the most logic-dense stage:
  - 55a: statements + expressions (the bulk; `if`/loops/`let`/calls/binops)
  - 55b: classes + dispatch (slot access, generic call lowering)
  - 55c: closures + blocks (cell conversion, non-local exit)
  each oracle-gated against the Rust lowering before moving on.
- **Flag:** `--lower-with-dylan` / `NOD_LOWER_WITH_DYLAN`.

## Sprint 56 — consolidation + flip to default (the payoff)

Today each stage crosses the Dylan↔Rust shim boundary *separately*,
re-encoding wire formats per file (lex→parse→expand→sema→lower = five
round-trips, each re-lexing the stdlib). That is the **~50× slowdown** we
deliberately deferred. Sprint 56 collapses the chain into **one Dylan
front-end that hands a single DFM module** to the Rust back-end:

- Retire the per-stage `--…-with-dylan` flags into the default path; Dylan
  becomes the only front-end.
- **The deferred perf returns here** — not via hybrid tuning, but because
  the per-stage round-trips vanish (one wire crossing at the DFM boundary,
  the stdlib lexed once). The locked decision ("perf waits for the single
  DFM handoff") cashes out.
- **Gate:** full `cargo test` green with the Dylan front-end as default;
  the bench suite's regression guards (`ratio >= 0.95`) hold against the
  pre-consolidation baseline.
- **Milestone:** *front-end fully self-hosted.* The only Rust left in the
  compile path is the part that should always have been Rust — the
  machine-code generator, the collector, the linker.

## The shape of it

**54 makes sema real, 55 makes lowering Dylan, 56 makes it one thing and
fast.** The on-ramp (finish 53's walk + kill the class-id drift) is the
foundation the three rungs stand on — skip it and 54/55 build on sand.

## Where it leaves us (when 56 lands)

NewOpenDylan compiles its own front-end with its own back-end. Year-3
self-hosting — the whole point of the arc that began with the Dylan lexer
in Sprint 45 — is complete. After that, growth is language-surface work
(more of the DRM in `nod-dylan/stdlib.dylan` macros) and back-end depth
(GC `gc.statepoint`, AOT breadth), not front-end migration.
