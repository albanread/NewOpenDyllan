# `stdlib/` — the Dylan standard library

The NewOpenDylan standard library, written in Dylan. These `.dylan` files are
**embedded into the compiler at build time** (via `include_str!` in
[`src/nod-sema/src/stdlib.rs`](../src/nod-sema/src/stdlib.rs)) and loaded once
per process by `nod_sema::stdlib::ensure_loaded()` — they are not separate
files at runtime.

## Load order

The loader processes the files in the order listed in the `STDLIB_FILES`
constant in `src/nod-sema/src/stdlib.rs`. `macros.dylan` must stay **first**
(it owns every stdlib `define macro`, surfaced to the parser/expander via
`stdlib_macro_source()`); the rest contribute their definitions to the merged
stdlib module in listed order. **Adding a new file: drop it here AND add an
entry to `STDLIB_FILES`** (both the `include_str!` path and the label) — there
is no auto-discovery; the constant is the manifest.

Each top-level `define function` is rewritten by the loader into a
single-method generic on `<object>` so user code can reach it through the
process-global dispatch table.

## Files

| file | contents |
|------|----------|
| `macros.dylan` | all stdlib `define macro`s (`for-each`, `unless`, `when`, `cond`, `iterate`, `test`/`suite`, …) |
| `collections.dylan` | `<collection>` family, `reduce`/`map`, `check-*` helpers |
| `arrays.dylan` | array classes/ops |
| `system-classes.dylan` | DRM class markers |
| `strings.dylan` | string + `<character>` ops |
| `sequences.dylan` | sequence ops (`member?`, `count`, `any?`, `every?`, …) |
| `lists.dylan` | list/sequence helpers (`nth`, `take`, `drop`, `sum`, `zip`, …) |
| `functional.dylan` | `identity`, `complement`, `compose`, `curry`/`rcurry`, `apply`, `always`, `disjoin`/`conjoin` |
| `numbers.dylan` | numeric ops (`abs`, `gcd`, `modulo`, `logand`, …) |
| `ffi-callbacks.dylan` | FFI callback glue |
| `structs.dylan` | struct accessors |
| `streams.dylan` | stream classes/ops |
| `win32-constants.dylan` | **auto-generated** — do not hand-edit; regenerated by `cargo run -p nod-winapi --bin generate_constants` from `data/win32_constants.txt` |

## Policy

**New stdlib API lands HERE by default**, not in `src/nod-runtime/` (Rust). The
boundary is defined by [`docs/STDLIB_BOUNDARY.md`](../docs/STDLIB_BOUNDARY.md):
write the Dylan version first; only fall back to Rust for a legitimate
primitive (GC, safepoints, FFI/OS, tag/layout, atomics, bootstrap). When
lifting code from Open Dylan, see
[`docs/UPSTREAM_OPENDYLAN.md`](../docs/UPSTREAM_OPENDYLAN.md) for attribution.
